# Initial Overlap Topics

## 1. Final Entity Definitions
Agree on:
- exact table names
- exact column names
- primary keys
- foreign keys
- null vs not null
- enum/categorical values

Example:
- energy_level: low / medium / high
- goal_status: active / completed / paused
- meal_type: breakfast / lunch / dinner / snack

## 2. Naming Conventions
Pick one style and stick to it:
- `snake_case` everywhere
- singular vs plural table names
- route naming
- template naming

Decision notes (`I'd do`):
- table names: singular (`user`, `goal`) or plural (`users`, `goals`) — just pick one
- Python files: `snake_case`
- API routes: plural is usually cleaner (`/goals`, `/metrics`)

## 3. Auth / Data Ownership Rules
Agree that:
- every user-owned record has `user_id`
- every fetch/update/delete must be scoped to the logged-in user
- no route trusts a `user_id` from the client

## 4. MVP Scope
Decide what must work first:
- register/login/logout
- dashboard
- daily metrics CRUD
- goals CRUD
- workouts CRUD
- nutrition CRUD
